1) Resumen para entender el proyecto en 60 segundos

tratohechoSJ es un marketplace local. En palabras simples: una web donde una persona publica algo para vender y otra persona lo encuentra rapido y le escribe por WhatsApp.

Idea central: "menos pasos, mas contacto real".
  • Explorar publicaciones sin registrarse.
  • Publicar solo si hay sesion iniciada.
  • Contactar al vendedor con un boton de WhatsApp.
  • Version web activa y wrapper Android en avance.

2) Tecnologias, explicadas sin tecnicismos

Next.js 16 + React 19 + TypeScript + Tailwind:

  • Next.js: organiza la web por paginas y rutas.
  • React: permite armar bloques reutilizables (botones, cards, formularios).
  • TypeScript: evita errores al programar, como un corrector inteligente.
  • Tailwind: acelera el estilo visual sin escribir CSS gigante.

Supabase (Auth + Postgres + Storage):

  • Auth: login y registro.
  • Postgres: guarda publicaciones (titulo, precio, descripcion, etc).
  • Storage: guarda imagenes de las publicaciones.
  • RLS: reglas de seguridad para que cada usuario solo edite lo suyo.

Capacitor: envuelve la web en una app Android.

  • No se reprograma todo en otro framework.
  • Se reutiliza la base web ya construida.
  • Estado actual: configurado, con pendiente de build APK debug final por entorno Java/JDK.

3) Arquitectura en modo visual

El sistema esta pensado para ser simple y claro. Primero se muestra contenido publico, y solo cuando alguien quiere publicar, se exige iniciar sesion.

Usuario abre la web
Rutas publicas: Feed y detalle
Si quiere publicar: Login
Guardar post en Supabase
Imagen al bucket post-images
Comprador contacta por WhatsApp

Rutas publicas: / y /post/[id]. Rutas privadas: /create-post, /my-posts, /my-posts/[id]/edit. La proteccion se resuelve en AppShell.

4) Como se programo (metodo de trabajo)

Paso 1 - Base funcional minima

Primero se hizo el flujo principal: ver publicaciones, crear publicaciones y guardar imagenes.

Paso 2 - Seguridad y permisos

Se agregaron politicas RLS en Supabase para que cada persona solo pueda modificar lo suyo.

Paso 3 - Experiencia de usuario

Se mejoro el uso en mobile, se agregaron skeletons, estados vacios y navegacion simple.

Paso 4 - Conversion real

Se priorizo WhatsApp para acortar el camino desde ver un producto hasta hablar con el vendedor.

Paso 5 - Escalado sin romper

Se dejo lista la base para Android con Capacitor, sin rehacer toda la aplicacion.

6) Base de datos y seguridad para no programadores

Imagina una planilla central llamada posts. Cada fila es una publicacion. Ademas, hay una "bodega" de imagenes llamada post-images.

  • Tabla posts: titulo, descripcion, precio, categoria, whatsapp, imagen, fecha y usuario dueno.
  • RLS activo: cualquiera puede ver; solo el dueno puede editar o borrar su publicacion.
  • Storage: las imagenes se suben con reglas que respetan la cuenta autenticada.
Resultado: privacidad y orden sin hacer una arquitectura pesada.

7) Mini guia para practicar

Si luego quieres verlo en accion:

  1. Abrir una terminal en la carpeta del proyecto.
  2. Instalar dependencias: npm install.
  3. Levantar entorno local: npm run dev.
  4. Abrir navegador en http://localhost:3000.
  5. Probar: explorar, login, crear publicacion, contactar por WhatsApp.
